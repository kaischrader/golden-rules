Input Dialogs
Input dialogs let the user enter or change some data.
  1. The application is grayed out in the background as soon as a dialog pop up window is opened.
  2. The dialog pop up can be closed by clicking [X], the [ESC]-button on the keyboard or the [Cancel]-button in the pop up, without taking over any of the changes performed by the user.
    1. If no action was performed by the user, the pop up window can also be closed by clicking into the grayed out background.
    2. If the user has performed at least one action and accidentally clicks into the grayed out background, the pop up may not be closed.
  3. Required fields are clearly and consistently marked as mandatory/required.
  4. Two types of hints:
    1. Assistance: For correct input of required data in a neutral font color (i.e. pre filled fields with example input)
    2. Error hint: For data input in an invalid form (immediate feedback needed) made visible in a clearly emphasized font color. Error hints have to be verbalized in a clearly understandable manner and have to give the user a helpful guidance to solve the error case
  5. To complete the pop up action, the [Apply] button must be clicked. This button is only clickable when all required fields are filled out in a correct manner and when all potential errors are solved.
  6. Congruent/coincided wording is needed for both, pop up title and text on [Apply] button. Example: Title of dialog: Create new Label, text on [Apply] button: Create Label
  7. Buttons can be categorized in types. Symbols are attached to these types with a specific color for an intrinsic and fast recognition of the button function.
  8. Rule of Thumb: Dialog pop up consists of not more than five to six elements in total and does not include multiple steps
    1. Size: Maximum 50% of the whole parent element
    2. Position: In the upper half or centered in relation to the parent element
  9. Do not use cascading/overlapping dialogs. Dialogs inside of dialogs should be avoided as much as possible.
  10. Avoid flickering dialogs. These are dialogs which change their size depending on the content.
